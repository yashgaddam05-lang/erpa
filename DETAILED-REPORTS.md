## 2.1 Structural SEO Changes

### Hub-and-Spoke Architecture
The most important SEO change. The old site had a flat structure where service pages existed in isolation. The rebuilt site uses a **hub-and-spoke model** where:

**6 Hub Pages** serve as pillar content for primary keyword clusters:
1. `/solutions/workday` → targets "Workday consulting services"
2. `/solutions/aws` → targets "AWS ERP cloud solutions"
3. `/solutions/peoplesoft` → targets "PeopleSoft managed services"
4. `/industries/higher-education` → targets "higher education ERP"
5. `/industries/government` → targets "government ERP modernization"
6. `/industries/healthcare` → targets "healthcare ERP consulting"

**7 Spoke Pages** target specific long-tail keywords and link back to their hub:
- `/solutions/workday/implementation` → "Workday implementation services"
- `/solutions/workday/managed-services` → "Workday managed services AMS"
- `/solutions/workday/optimization` → "Workday optimization integrations"
- `/solutions/aws/migration` → "AWS cloud migration ERP"
- `/solutions/aws/cloud-managed-services` → "cloud managed services"
- `/solutions/peoplesoft/managed-services` → "PeopleSoft managed services"
- `/solutions/peoplesoft/modernization` → "PeopleSoft modernization AWS"

**Why this matters:** Google rewards topical authority. A hub page with 3 detailed spoke pages signals deeper expertise than a single thin page. Each spoke page ranks independently for its long-tail keyword while passing link equity up to the hub.

### URL Structure
Every URL is clean, keyword-rich, and hierarchical:
- `/solutions/workday/implementation` (not `/workday-solutions/implementation-services/`)
- `/industries/higher-education` (not `/higher-education/`)
- `/resources/case-studies` (not `/case-study/`)

### Breadcrumb Navigation
Every inner page has breadcrumbs:
```
Home / Solutions / Workday / Implementation
```
This creates both a **user navigation aid** and **structured data signal** for Google.

Every page is **statically generated at build time** (indicated by the `○` symbol in the build output). This means:
